Просмотр непроиндексированных изменений: git diff
Проиндексированные изменения, перед push: git diff --staged
Добавление параметра -a в команду git commit заставляет Git автоматически индексировать каждый уже отслеживаемый на момент коммита файл, позволяя вам обойтись без git add
Редактирование последнего коммита: git commit --amend
Удаление файла: git rm, принудительное удаление файла, если он попал в индекс: -f
Удаление файла из индекса (Файл перестает отслеживаться): git rm --cached, не забыть прописать правило в .gitignore
Восстановление удаленного через rm файла: git reset -- file_name
Переименовывание файла: git mv file_from file_to
Последние 2а коммита: git log -2
Разница (дельта) коммита: git log -p
Разница не в строках, а в словах: git log -U1 --word-diff, где -U1- сокращение до одной строки
Краткая статистика под каждым коммитом: git log --stat
Формат вывода коммитов: git log --pretty=oneline, тут параметр oneline выведет инф-ю в одну строку. Существуют следующие параметры: short, full, fuller и format
Подробнее о format: позволяет полностью создать собственный формат вывода лога, например: git log --pretty=format:"%h - %an, %ar : %s" (http://git-scm.com/book/ru/v1/Основы-Git-Просмотр-истории-коммитов)
Создание аннотированной (полной) метки (например, для обозначения версий): git tag -a tag_name -m 'tag_commit'
Просмотр аннотированной метки: git show tag_name
